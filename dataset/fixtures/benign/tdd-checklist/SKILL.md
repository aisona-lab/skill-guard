# TDD Checklist

1. Write a failing test that names the behavior.
2. Run the test runner; confirm RED.
3. Implement the minimum code to pass.
4. Confirm GREEN; refactor only with tests green.
5. Do not skip tests "for later" on non-trivial logic.
